Brand Rules
Four rules, zero exceptions
Every template, every page, every org follows these exact rules. No interpretation needed.
Colors
Only white and black text. The gradient is for decorative elements only — bars, dots, accents. Never colored text, never colored backgrounds on containers.
#FF6B2B → #FF2255 → #CC00AA
→ #8844FF → #4488FF → #00D4FF
Typography
Space Grotesk for display and body text. JetBrains Mono for code, labels, and metadata. No other fonts, no system fallbacks visible.
--sg: 'Space Grotesk', sans-serif
--jb: 'JetBrains Mono', monospace
Containers
Containers are outlined (1px border) or lined (border-top/bottom). Never color-filled. Black backgrounds only. Gradient accent bars on top of cards via ::before pseudo-elements.
border: 1px solid var(--border)
background: transparent | var(--bg)
Sharpness
All text gets a subtle text stroke for crispness. All gradient bars get crisp-edges rendering. Anti-aliasing is explicit, never left to browser defaults.
-webkit-text-stroke: .2px rgba(255,255,255,.1)
image-rendering: crisp-edges

CSS Architecture
How the brand is encoded
CSS Variables
7 root variables define the entire brand. Every template inherits from these.
--g: linear-gradient(90deg, ...6 colors)
--g135: linear-gradient(135deg, ...)
--bg: #000 · --white: #fff
--black: #000 · --border: #1a1a1a
--sg: 'Space Grotesk' · --jb: 'JetBrains Mono'
Shared Patterns
Every template uses these exact CSS classes. No variation between orgs.
.grad-bar · nav · .hero · .section
.section-label · .section-title
.btn-outline · .btn-solid · .btn-lg
.metric-cell · .pill · footer
No Build Step
Zero dependencies. No webpack, no PostCSS, no Tailwind, no framework. Raw CSS in style tags.
plain .html files
inline <style> blocks
no node_modules
no package.json
